# 🏥 Smart Healthcare Monitoring System (SHMS)

## 📌 Problem Statement
Modern healthcare lacks continuous monitoring for patients with chronic illnesses like diabetes, hypertension, and heart diseases. This often leads to delayed diagnosis and increased health risks.

---

## ⚠️ Challenges
- Delayed detection of critical conditions  
- High healthcare costs  
- Frequent hospital visits  
- Lack of real-time updates for caregivers  

---

## 💡 Proposed Solution
A Smart Healthcare Monitoring System using IoT wearable devices and a mobile application that:

- Monitors vital signs (Heart Rate, Blood Pressure, Glucose Levels)  
- Sends real-time alerts 🚨  
- Provides AI-based health insights  
- Enables remote monitoring by doctors  

---

## 👩‍💻 My Role
**Mobile Application Developer**

I analyzed user needs and designed application requirements based on different stakeholders.

---

## 📊 User Requirement Table

| Role | Pain Points | Requirements | Features |
|------|------------|-------------|----------|
| Patient | Difficult to track health | Easy UI, real-time alerts | Dashboard, reminders |
| Doctor | No continuous data | Access to patient history | Graphs, alerts |
| Caregiver | Anxiety, no updates | Live monitoring | Notifications, SOS |
| Admin | High cost, inefficiency | Centralized system | Reports, analytics |
| Developer | Integration issues | Secure & scalable system | Cloud, APIs |
